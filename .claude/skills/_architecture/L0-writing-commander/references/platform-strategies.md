# 平台策略详解

## 概述

本文档详细描述各主流内容平台的特性、算法偏好、内容策略和最佳实践。
writing-commander 和 platform-adapter 通过查阅此文件选择最优平台策略。

---

## 小红书 (Xiaohongshu)

### 平台特性

| 维度 | 描述 |
|------|------|
| 用户画像 | 以女性为主，年轻群体，追求品质生活 |
| 内容调性 | 种草、分享、真实体验 |
| 核心价值 | "标记我的生活" |
| 算法偏好 | 互动率、完播率、收藏率 |

### 内容规范

```yaml
标题:
  长度: ≤20字
  风格: 口语化、emoji开头
  公式:
    - "🔥+关键词+悬念/利益点"
    - "数字+类型词+价值点"
  示例:
    - "🔥 学霸都在用的学习法 | 后悔没早知道"
    - "5个让皮肤变好的习惯｜亲测有效"

正文:
  长度: 500-1000字最佳
  结构:
    - 开头3行决定是否继续看
    - 分段落，每段2-4行
    - 重点信息用 emoji 标记
  emoji密度: 每100字2-4个

话题标签:
  数量: 3-8个
  策略:
    - 热门话题 30% (获取曝光)
    - 精准话题 50% (精准触达)
    - 品牌话题 20% (建立认知)
```

### 爆款要素

1. **首图** - 清晰、有吸引力、与内容相关
2. **标题** - 引发好奇或提供明确价值
3. **前3行** - 必须抓住注意力
4. **干货密度** - 有实质内容，而非空洞描述
5. **互动引导** - 末尾引导评论、收藏

### 内容类型表现

| 类型 | 互动率 | 适用场景 |
|------|--------|---------|
| 教程攻略 | 高 | 技能分享 |
| 真实体验 | 高 | 产品评测 |
| 清单合集 | 中高 | 资源汇总 |
| 个人故事 | 中 | 情感共鸣 |
| 观点输出 | 中 | 讨论话题 |

### 发布时间建议

- **工作日**: 12:00-14:00, 18:00-22:00
- **周末**: 10:00-12:00, 15:00-22:00
- **最佳**: 周末晚上 20:00-21:00

## 微信公众号 (WeChat Official Account)

### 平台特性

| 维度 | 描述 |
|------|------|
| 用户画像 | 广泛，各年龄段，职场人士偏多 |
| 内容调性 | 深度、专业、有观点 |
| 核心价值 | 订阅制深度阅读 |
| 算法偏好 | 打开率、阅读完成率、分享率 |

### 内容规范

```yaml
标题:
  显示长度: 约22字（朋友圈显示）
  最大长度: 64字
  风格: 金句感、引发好奇
  公式:
    - "痛点+解决方案"
    - "数字+价值承诺"
    - "悬念/反常识+答案暗示"
  示例:
    - "为什么别人学得快？这5个方法是关键"
    - "3年从月薪5000到年入百万，我只做对了一件事"

正文:
  长度: 1500-3000字
  结构:
    - 开篇: 故事/观点引入 (300字)
    - 正文: 小标题分段，每段300-500字
    - 结尾: 总结+引导互动 (200字)
  金句处理: 加粗、引用框、方便截图

引导互动:
  文末三件套:
    - "点赞" - 简单认可
    - "在看" - 分享到看一看
    - "分享" - 转发朋友圈
```

### 爆款要素

1. **标题** - 一句话说清价值和吸引力
2. **开篇** - 前100字决定是否继续
3. **金句** - 可截图分享的句子
4. **逻辑性** - 结构清晰，论证有力
5. **分享动机** - 让读者有理由转发

### 内容类型表现

| 类型 | 分享率 | 适用场景 |
|------|--------|---------|
| 深度分析 | 高 | 行业洞察 |
| 方法论 | 高 | 技能提升 |
| 人物故事 | 中高 | 励志/启发 |
| 观点文 | 中 | 热点评论 |
| 盘点合集 | 中 | 资源汇总 |

### 发布时间建议

- **工作日**: 7:00-9:00 (通勤), 12:00-13:00 (午休), 20:00-22:00 (晚间)
- **周末**: 9:00-11:00, 20:00-22:00
- **最佳**: 周二-周四 早8点

## Twitter/X

### 平台特性

| 维度 | 描述 |
|------|------|
| 用户画像 | 全球化，科技/媒体/政治关注者 |
| 内容调性 | 简洁、有力、即时 |
| 核心价值 | 实时信息和观点交流 |
| 算法偏好 | 互动率、引用转发、停留时间 |

### 内容规范

```yaml
单条推文:
  长度: ≤280字符
  风格: 简洁有力、观点鲜明
  公式:
    - "观点 + 论据/例证"
    - "问题 + 简短回答"
  示例:
    - "最被高估的生产力技巧：多任务处理。研究表明，任务切换会让效率降低40%。"

Thread 结构:
  长度: 3-10条
  结构:
    1/ 钩子（最重要！）- 引发阅读欲望
    2-4/ 背景/问题描述
    5-8/ 核心观点/方法
    9/ 总结
    10/ CTA + 转发邀请
  每条: 保持独立可读

话题标签:
  数量: 1-3个
  策略: 精准为主，避免过多
```

### 爆款要素

1. **第一条** - 必须吸引人展开 Thread
2. **观点鲜明** - 有立场，不模棱两可
3. **信息密度** - 每条都有价值
4. **结构清晰** - 编号、格式整齐
5. **互动设计** - 提问、投票、引发讨论

### Thread 最佳实践

```
开头模板:
- "I've spent X years doing Y. Here's what I learned:"
- "Unpopular opinion: [观点]"
- "Most people get [主题] wrong. Here's why:"
- "[数字] lessons from [经历/研究]"

结尾模板:
- "If you found this helpful, please RT the first tweet."
- "Follow me for more [主题] content."
- "What's your take? Reply below."
```

### 发布时间建议 (UTC)

- **全球最佳**: 15:00-17:00 UTC
- **美国为主**: 13:00-15:00 UTC (美国早晨)
- **亚洲为主**: 1:00-3:00 UTC (亚洲晚间)

## 跨平台分发策略

### 同一内容的平台适配

```
原始长文 (2000字深度文章)
├── 公众号: 原文发布，添加引导语
├── 小红书: 提炼5-7个核心点，口语化改写
└── Twitter: 拆分为 Thread，每条一个核心观点
```

### 分发顺序建议

1. **首发平台** - 选择最匹配的平台
2. **24小时内** - 适配发布到其他平台
3. **注意去重** - 避免完全相同内容
