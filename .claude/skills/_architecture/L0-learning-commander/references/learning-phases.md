# 学习阶段详解

## 四阶段学习框架

```
┌─────────────────────────────────────────────────────────────┐
│                    学习四阶段模型                             │
├─────────────────────────────────────────────────────────────┤
│  Phase 1        Phase 2        Phase 3        Phase 4       │
│  ┌─────┐       ┌─────┐       ┌─────┐       ┌─────┐         │
│  │消化 │  ──▶  │结构化│  ──▶  │内化 │  ──▶  │应用 │         │
│  │Digest│       │Structure│    │Internalize│  │Apply│        │
│  └─────┘       └─────┘       └─────┘       └─────┘         │
│     ▲                            │                          │
│     └────────────────────────────┘                          │
│            补救循环（验证失败时）                              │
└─────────────────────────────────────────────────────────────┘
```

---

## Phase 1: 消化 (Digest)

### 目标
将外部知识转化为内部表征

### 编排器
`learning-orchestrator`

### 调用的 Skills
| Skill | 职责 |
|-------|------|
| content-curator | 策展优质学习资源 |
| knowledge-extractor | 提取结构化知识项 |
| analogy-explainer | 生成易懂类比解释 |

### 流程
```
Step 1: 内容获取
├── 调用 content-curator
├── 收集相关资源
└── 按质量排序

Step 2: 知识提取
├── 调用 knowledge-extractor
├── 提取概念、原则、事实
└── 构建初步层次结构

Step 3: 类比生成
├── 调用 analogy-explainer
├── 为核心概念生成类比
└── 标注类比局限性

Step 4: 质量检查
├── 验证知识覆盖度
└── 确保结构完整性
```

### 产出
- 结构化知识项列表
- 概念摘要
- 类比解释集合
- 初步概念层次

### 质量标准
| 指标 | 阈值 |
|------|------|
| 核心概念覆盖 | ≥80% |
| 每个概念有定义 | 100% |
| 核心概念有类比 | ≥60% |

---

## Phase 2: 结构化 (Structure)

### 目标
构建知识间的关联结构

### 编排器
`knowledge-graph-orchestrator`

### 调用的 Skills
| Skill | 职责 |
|-------|------|
| spatial-mapper | 概念空间映射 |
| diagram-generator | 生成可视化图表 |

### 流程
```
Step 1: 关系增强
├── 补充隐含关系
└── 验证关系合理性

Step 2: 空间映射
├── 调用 spatial-mapper
└── 确定节点位置

Step 3: 聚类分析
├── 识别概念群组
└── 命名聚类

Step 4: 可视化生成
├── 调用 diagram-generator
├── 生成 Mermaid 代码
└── 生成多种格式

Step 5: 学习路径推导
├── 拓扑排序
└── 生成建议学习顺序
```

### 产出
- 知识图谱（节点+边）
- 概念聚类
- Mermaid/Graphviz 代码
- 建议学习顺序

### 质量标准
| 指标 | 阈值 |
|------|------|
| 图谱连通性 | 100% |
| 无循环依赖 | 100% |
| 有明确中心概念 | 是 |

---

## Phase 3: 内化 (Internalize)

### 目标
通过验证确保深度理解

### 编排器
`verification-orchestrator`

### 调用的 Skills
| Skill | 职责 |
|-------|------|
| self-explanation-validator | 验证自我解释 |
| socratic-questioner | 苏格拉底式提问 |
| analogy-explainer | 澄清误解时使用 |

### 流程
```
Step 1: 自我解释验证
├── 要求学习者解释核心概念
├── 调用 self-explanation-validator
└── 识别误解和遗漏

Step 2: 苏格拉底对话
├── 调用 socratic-questioner
├── 通过提问引导深入思考
└── 自适应调整问题难度

Step 3: 类比检验
├── 要求学习者解释类比
├── 检验类比理解的准确性
└── 使用 analogy-explainer 澄清

Step 4: 掌握度评估
├── 综合各项验证结果
├── 计算掌握度分数
└── 确定掌握等级

Step 5: 补救建议
├── 识别知识差距
└── 生成补救计划
```

### 产出
- 掌握度评估报告
- 误解纠正记录
- 苏格拉底对话记录
- 补救建议（如需要）

### 质量标准
| 指标 | 阈值 |
|------|------|
| 自解释覆盖 | ≥80% 核心概念 |
| 严重误解已纠正 | 100% |
| 掌握度达标 | ≥目标阈值 |

### 验证失败处理
- 返回 Phase 1 补充知识
- 返回 Phase 2 加强结构理解
- 降低目标等级

---

## Phase 4: 应用 (Apply)

### 目标
通过实践巩固学习

### 编排器
`micro-project-orchestrator`

### 调用的 Skills
| Skill | 职责 |
|-------|------|
| code-gen | 生成练习代码框架 |
| test-gen | 生成测试用例 |
| code-review | 评审学习者代码 |

### 流程
```
Step 1: 项目生成
├── 根据学习内容设计微项目
├── 确定难度级别
└── 生成项目骨架

Step 2: 练习指导
├── 提供分步指导
├── 给出提示和参考
└── 允许查阅学习材料

Step 3: 代码评审
├── 调用 code-review
├── 检查概念应用正确性
└── 提供改进建议

Step 4: 能力评估
├── 评估应用能力
├── 验证知识迁移
└── 确定最终掌握等级
```

### 产出
- 完成的微项目代码
- 代码评审报告
- 最终能力评估
- 后续学习建议

### 质量标准
| 指标 | 阈值 |
|------|------|
| 项目完成度 | ≥80% |
| 概念应用正确 | ≥70% |
| 测试通过率 | ≥60% |

```
  │            │            │
  │            │            ▼
  │            └───────── 补救循环
  │                         │
  └─────────────────────────┘
```

## 跳过阶段条件

| 跳过阶段 | 条件 |
|---------|------|
| Phase 2 | 目标等级 = awareness |
| Phase 3 | 目标等级 ≤ understanding 且学习者已有基础 |
| Phase 4 | 目标等级 < mastery |
